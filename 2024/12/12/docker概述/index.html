DOCKER简述

Docker 是一个开源的平台，用于自动化开发、部署和运行应用程序。它通过利用操作系统级别的虚拟化技术（容器化）来提供轻量级的、可移植的软件包，这些软件包包含了应用程序及其所有依赖项。Docker 使得开发者可以在任何环境中一致地运行他们的应用，无论是在开发、测试还是生产环境中。

docker组件

Dockerfile

Dockerfile 是一个文本文件，包含了用于构建 Docker 镜像的一系列指令。通过 Dockerfile，你可以定义应用程序及其所有依赖项的环境，确保它们可以在任何支持 Docker 的环境中一致地运行。Dockerfile 是实现容器化应用的基础，它使得自动化构建和分发镜像成为可能

容器映像

容器映像是一个轻量级、独立的、可执行的软件包，它包含了运行某个软件所需的所有内容：代码、运行时环境、系统工具、库和设置。 使用docker images 来查看所有本地镜像

容器

容器是容器映像的一个运行实例。当从映像启动容器时，它会创建一个独立的、隔离的运行环境，在其中执行应用程序。

使用docker ps可以查看所有正在运行的容器，如果想要查看所有容器(正在运行和已经停止的)可以使用docker ps -a命令

docker官网的定义

容器是软件的标准单元，它将代码及其所有依赖项打包在一起，以便应用程序能够快速可靠地从一个计算环境运行到另一个计算环境。Docker 容器映像是一个轻量级、独立的可执行软件包，其中包含运行应用程序所需的一切：代码、运行时、系统工具、系统库和设置。

容器镜像在运行时成为容器，而对于 Docker 容器而言，镜像在Docker Engine上运行时成为容器。容器化软件适用于基于 Linux 和 Windows 的应用程序，无论采用何种基础设施，其运行方式都相同。容器将软件与其环境隔离，并确保其尽管存在开发和暂存之间的差异，但仍能统一运行。

如果你学习过java，你可以把容器映像和容器的关系视为类和实例的关系，通过容器映像可以创建多个容器

命名空间

Docker 命名空间是 Linux 的一项基本功能，Docker 使用它来为容器创建隔离环境。

它们通过创建全局系统资源的单独实例来提供一层隔离，使每个容器都认为自己拥有自己独特的一组资源。Docker 使用多种类型的命名空间，包括 PID（进程 ID）、NET（网络）、MNT（挂载）、UTS（Unix 分时系统）、IPC（进程间通信）和 USER 命名空间，通过利用这些命名空间，Docker 可以创建轻量级、可移植且安全的容器，这些容器可以在不同的环境中一致运行。

docker的安装

具体的下载流程请访问获取 Docker_Docker中文网这里不详细解释

docker engine(Linux)

Docker Engine 特指 Docker Desktop 组件的一个子集，它是免费的开源组件，只能安装在 Linux 上。Docker Engine 可以构建容器映像、从中运行容器，并且通常可以执行 Docker Desktop 可以执行的大部分操作，但它仅适用于 Linux，并且无法提供 Docker Desktop 提供的所有开发人员体验

docker desktop(Win,Max,Linux)

Docker Desktop 是一款易于安装的应用程序，可让开发人员在其台式机上快速设置 Docker 环境。它适用于 Windows 和 macOS 操作系统。Docker Desktop 旨在简化管理和运行 Docker 容器的过程，提供用户友好的界面并与主机操作系统无缝集成

docker中的数据持久性

Docker 可运行与主机操作系统分离的独立代码片段（包括应用程序及其依赖项）容器。容器终止后，存储在容器中的任何数据都将丢失

卷挂载(volumes mounts)

卷装载是一种将主机系统上的文件夹或文件映射到容器内的文件夹或文件的方法。这样，即使容器被移除，数据仍会保留在容器外部。此外，多个容器可以共享同一个卷，从而轻松实现容器之间的数据共享。

绑定挂载(bind mounts)

使用绑定挂载时，主机上的文件或目录将挂载到容器中。文件或目录由其在主机上的绝对路径引用。相比之下，使用卷时，会在主机上的 Docker 存储目录中创建一个新目录，Docker 会管理该目录的内容。
