JDBC 与ORM

JDBC

JDBC的缺点：需要手动的完成面向对象的Java语言、面向关系的数据库之间数据的转换，代码繁琐无技术含量，影响了开发效率。

如图所示，查询是需要手动的将结果集的列数据转换为Java对象的属性；而添加操作时需要手动将Java对象的属性转换为数据库表的列字段。

ORM

ORM，Object-Relationl Mapping，对象关系映射，它的作用是在关系型数据库和对象之间作一个映射，这样我们在具体的操作数据库的时候，只要像平时操作对象一样操作它就可以了，ORM框架会根据映射完成对数据库的操作，就不需要再去和复杂的SQL语句打交道了。

ORM的持久化和持久层概念

  • 持久化

持久化就是把数据永久的保存到存储设备中（如磁盘）。持久化是将数据保存到关系型数据库、磁盘文件、XML数据文件

  • 持久层

持久层是将数据使用者和数据实体关联。之前访问JDBC访问数据库DAO层，后面采用MyBatis访问的数据库的Mapper层，就是持久层

Mybatis

Mybatis就是持久层的半自动化的ORM映射框架。MyBatis 可以使用简单的 XML 或注解来配置和映射原生信息，将接口和 Java 的 POJOs(Plain Ordinary Java Object,普通的 Java对象)映射成数据库中的记录。

MyBatis是一个半自动ORM框架，其本质是对JDBC的封装。使用MyBatis重点需要程序员编写SQL命令，不需要写一行JDBC代码

动态SQL

在 MyBatis 中，动态 SQL 是一个非常强大的特性，它允许你根据条件动态地构建 SQL 语句。以下是各个常用标签的使用场景和功能介绍：

1. if 标签

使用场景：当需要根据某些条件来决定是否包含某个 SQL 片段时。

  • 功能：<if> 标签用于条件判断，只有当 test 属性中的表达式为 true 时，才会将该标签内的内容添加到最终生成的 SQL 语句中。

  • 示例：

2. where 标签

使用场景：简化 WHERE 子句的构建，自动处理第一个条件前的 AND 或 OR。

  • 功能：<where> 标签会自动去除多余的 AND 或 OR，并且如果没有任何条件匹配，则不会生成 WHERE 子句。

  • 示例：

3. choose 标签

使用场景：实现类似 Java 的 switch-case 或者 if-else 的逻辑选择。

  • 功能：<choose> 标签配合 <when> 和 <otherwise> 使用，用来表示多条件分支的选择。当其中一个 <when> 条件满足时，其他条件将不再检查；如果没有一个 <when> 满足，则执行 <otherwise> 中的内容。

  • 示例：

4. set 标签

使用场景：用于构建 UPDATE 语句中的 SET 子句，自动处理逗号分隔符。

  • 功能：<set> 标签会自动去掉最后一个更新项后面的多余逗号，并且如果没有任何更新项，则不会生成 SET 子句。

  • 示例：

5. trim 标签

使用场景：灵活控制 SQL 片段的前后缀，常用于自定义复杂 SQL 结构。

  • 功能：<trim> 标签可以指定 prefix、suffix、prefixOverrides 和 suffixOverrides 属性，用于添加或移除特定的前缀或后缀。

  • 示例：

6. bind 标签

使用场景：创建变量绑定，通常与正则表达式一起使用来预处理参数值。

  • 功能：<bind> 标签可以在 SQL 内部创建一个新的变量，并将其绑定到当前上下文中。这对于复杂的查询特别有用，比如全文搜索。

  • 示例：

7. sql 标签

使用场景：复用 SQL 片段，避免重复代码。

  • 功能：<sql> 标签用于定义可复用的 SQL 片段，然后可以通过 <include> 标签引用这些片段。

  • 示例：

8. foreach 标料

使用场景：遍历集合类型（如 List、Set 等），适用于 IN 子句或其他需要迭代的情况。

  • 功能：<foreach> 标签可以迭代集合，并生成相应的 SQL 语法。常用的属性包括 collection（集合名称）、item（每次迭代的元素名称）、separator（元素之间的分隔符）以及 open 和 close（包围整个集合的符号）。

延迟加载

延迟加载，又称按需加载。延迟加载的内容等到真正使用时才去进行加载（查询）。多用在关联对象或集合中。

延迟加载的好处：先从单表查询、需要时再从关联表去关联查询，大大降低数据库在单位时间内的查询工作量,将工作在时间上的分配更加均匀，而且单表要比关联查询多张表速度要快。

延迟加载的设置

第一步：全局开关：在sqlMapConfig.xml中打开延迟加载的开关。配置完成后所有的association和collection元素都生效

lazyLoadingEnabled:是否开启延迟加载。是Mybatis是否启用懒加载的全局开关。当开启时，所有关联对象都会延迟加载。特定关联关系中可通过设置fetchType属性来覆盖该项的开关状态

aggressiveLazyLoading：当开启时，任何方法的调用都会懒加载对象的所有属性。否则，每个属性会按需加载,

第二步：分开关：指定的association和collection元素中配置fetchType属性。eager：表示立刻加载；lazy：表示延迟加载。将覆盖全局延迟设置。

注解开发

1.使用注解没有实现Java代码和SQL语句的解耦

2.无法实现SQL语句的动态拼接

3.进行多表的查询时定制ResultMap比较麻烦

缓存

一级缓存

一级存储是SqlSession上的缓存，默认开启，是一种内存型缓存,不要求实体类对象实现Serializable接口。

二级缓存

二级缓存是以namespace为标记的缓存，可以是由一个SqlSessionFactory创建的SqlSession之间共享缓存数据。默认并不开启。下面的代码中创建了两个SqlSession，执行相同的SQL语句，尝试让第二个SqlSession使用第一个SqlSession查询后缓存的数据。要求实体类必须实现序列化接口

三级缓存

分布式缓存框架：我们系统为了提高系统并发和性能，一般对系统进行分布式部署（集群部署方式）不适用分布缓存， 缓存的数据在各个服务单独存储，不方便系统开发。所以要使用分布式缓存对缓存数据进行集中管理.ehcache,redis ,memcache缓存框架。

Ehcache：是一种广泛使用的开源java分布式缓存。主要面向通用缓存，javaEE 和 轻量级容器。它具有内存和磁盘存储功能。被用于大型复杂分布式web application的
